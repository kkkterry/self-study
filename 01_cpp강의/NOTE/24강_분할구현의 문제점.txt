24강. 분할구현의 문제점
1. 중복정의의 문제
분할구현시 헤더 파일에 함수의 선언 및 정의가 되어있을경우 h파일을 참조하는 파일마다
함수가 선언 및 정의가 된다. 즉 중복정의의 문제가 생긴다.

그래서
헤더파일에는 함수의 선언만을 하고 함수의 구현은 헤더파일과 같은 이름을 가진 cpp파일에 한다.

func.h-> int add(int a, intb); 보통 함수 선언만 해둔다.

func.cpp -> #include "func.h"
	   int add(int a, int b){
	   	return a+b;
	   }

main.cpp -> #include <func.h>  -> int add(int a, intb); 
			        이고 아래 함수에 대한 정의가 되어있지 않지만 함수 활용가능
                                             (컴파일과정 링크과정에서 자동으로 됨; 어떤 과정인지는 명확지는 않음)

	    int main(){
	    	add(2,3);
		
		return 0;
	    }
